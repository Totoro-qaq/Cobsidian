# Note Types

Use the smallest note shape that preserves the knowledge.

## Learning Note

Use for concepts, tools, papers, videos, or conversations that teach a reusable topic.

Recommended sections:

- Summary
- Core Concepts
- Workflow or Mental Model
- Common Mistakes
- Related Notes

## Daily Capture Note

Use for quick daily capture when the material is useful but not ready for a full learning, project, or review note.

Recommended sections:

- Capture
- Useful Points
- Possible Links
- Follow-up

Keep it short. Prefer preserving retrieval value over polishing structure.

## Project Note

Use for a specific project, its architecture, implementation, or operations. Use a Review Note when retrospective analysis is the primary purpose.

Recommended sections:

- Context
- Goal
- Architecture or Implementation
- Operations
- Evidence
- Result
- Risks and Next Steps
- Related Notes

## Review Note

Use for incidents, failures, and experiments whose durable value comes from evidence-based learning and corrective action.

Recommended sections:

- Context
- Expected Outcome
- Actual Outcome
- Timeline
- Evidence
- Root Causes
- Corrective Actions
- Unresolved Questions
- Related Notes

## Dissection Note

Use for explaining why a specific product, technology, tool, repository, framework, prompt system, agent workflow, or codebase exists; how it creates outcomes; and how it differs across peers and time.

Recommended sections:

- Object and Scope
- Problem and Prior State
- Solution Thesis
- Mechanism Chain
- Adoption Effects
- Horizontal Comparison
- Vertical Evolution
- Distinctive Advantages
- Trade-offs and Limits
- Reusable Patterns
- Evidence and Open Questions
- Related Notes

Use `problem -> design choice -> mechanism -> outcome -> evidence` as the central chain. Horizontal comparison covers contemporary peers through shared dimensions; vertical evolution covers the predecessor, prior approach, or earlier version. Keep claimed, observed, inferred, and unknown outcomes distinguishable.

## Comparison Note

Use when the value comes from comparing options, versions, tools, or attempts.

Recommended sections:

- Short Conclusion
- Comparison Table
- Key Differences
- Decision Rules
- Related Notes

## Index Note

Use when multiple notes need a navigable entry point.

Recommended sections:

- Map
- Core Notes
- Project Notes
- Open Questions
