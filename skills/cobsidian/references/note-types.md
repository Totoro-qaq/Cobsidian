# Note Types

Use the smallest note shape that preserves the knowledge.

## Learning Note

Use for concepts, tools, papers, videos, or conversations that teach a reusable topic.

Recommended sections:

- Summary
- Core Concepts
- Workflow or Mental Model
- Common Mistakes
- Related Notes

## Daily Capture Note

Use for quick daily capture when the material is useful but not ready for a full learning, project, or review note.

Recommended sections:

- Capture
- Useful Points
- Possible Links
- Follow-up

Keep it short. Prefer preserving retrieval value over polishing structure.

## Project Note

Use for a specific project, experiment, incident, or implementation.

Recommended sections:

- Context
- Goal
- Implementation or Process
- Evidence
- Result
- Risks and Next Steps
- Related Notes

## Dissection Note

Use for reverse engineering an existing tool, repository, framework, prompt system, agent workflow, or codebase.

Recommended sections:

- Object of Study
- Purpose
- Entry Points
- Core Concepts
- Architecture or Flow
- Key Files or Components
- Reusable Patterns
- Limits and Open Questions
- Related Notes

Use this for source-code and workflow breakdowns such as agent frameworks, skill repositories, prompt systems, orchestration tools, and automation harnesses.

## Comparison Note

Use when the value comes from comparing options, versions, tools, or attempts.

Recommended sections:

- Short Conclusion
- Comparison Table
- Key Differences
- Decision Rules
- Related Notes

## Index Note

Use when multiple notes need a navigable entry point.

Recommended sections:

- Map
- Core Notes
- Project Notes
- Open Questions
