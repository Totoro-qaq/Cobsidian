# Markdown Style

## General

- Use one H1 title.
- Use stable H2 sections.
- Use tables for comparisons.
- Use fenced code blocks for commands, config, and logs.
- Keep raw transcripts out of the final note unless explicitly requested.

## Obsidian

- Use `[[wiki links]]` for internal notes.
- Avoid file extensions inside wiki links.
- Keep note titles human-readable.
- Use tags only when the existing vault uses tags consistently.

## Tone

Write for future retrieval:

- concise
- factual
- reusable
- not tied to one chat session unless it is a project log
