# Backlink Policy

Add links that help future retrieval.

## Good Links

- Core concepts mentioned in the note
- Parent index or learning path notes
- Closely related project notes
- Tool notes required to understand the workflow

## Bad Links

- Every repeated keyword
- Links to notes that only share one generic word
- Links to private records in a public/general note
- Broken links created without checking whether a note exists

## Practical Rule

Prefer 3 to 8 high-value links per note. Use more only for index notes.

## Related Notes Section

Use this when links are useful but would make the body noisy:

```markdown
## Related Notes

- [[Learning Topic]]
- [[Project Review]]
- [[Tool Name]]
```
