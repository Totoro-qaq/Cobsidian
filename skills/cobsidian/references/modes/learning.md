# Learning Mode

## When to Use

Use `learning` for concepts, courses, papers, videos, and ordinary explanations whose main value is understanding and future recall. Use `dissection` instead when the request is primarily about internal mechanics of a studied system.

## Required Inputs

Identify the topic, the material actually available, the learner's question, and the desired scope. Record source titles or locations only when the active host can read them.

## Default Knowledge Read

- Default depth: `standard`
- Default granularity: `single-note`

Keep the default evidence at `conversation` until source material is actually read or a verification result is obtained.

## Recommended Note Shape

Use a focused note with `Summary`, `Learning Goals`, `Core Concepts`, `Mental Model`, `Worked Example`, `Common Mistakes`, `Open Questions`, and `Related Notes`. Omit a section only when the material provides no durable content for it.

## Append, Single-Note, and Split Criteria

- **Append** when an existing note already explains the same concept and the new material adds evidence, examples, or corrections.
- **Single-note** when one coherent concept can be understood and retrieved without separate maintenance boundaries.
- **Split** when several independently reusable concepts need their own titles, backlinks, or update cycles; do not split merely because the source is long.

## Evidence Rules

Start at `conversation`. Upgrade to `source-grounded` only after reading the cited paper, video transcript, course material, file, or page. Use `verified` only when claims are additionally checked against runnable examples, tests, or authoritative results.

## Mode-Specific Validation

Verify that definitions are consistent, examples support the stated concept, unsupported claims are labeled, and every wiki link points to a confirmed note. Check that the note teaches rather than merely summarizes the conversation.

## Completion Report Additions

Report the learning scope, sections created or extended, evidence level, unresolved questions, and any source that could not be read or verified.
