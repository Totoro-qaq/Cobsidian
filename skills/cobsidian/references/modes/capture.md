# Capture Mode

## When to Use

Use `capture` for low-friction preservation when the user wants useful material recorded quickly before deeper organization. Keep it concise and never expand it to `deep` by default, even when the source is long.

## Required Inputs

Identify the material to preserve, a retrievable topic or date, the minimum useful context, and any explicit follow-up. Do not require research or a complete taxonomy before capture.

## Default Knowledge Read

- Default depth: `capture`
- Default granularity: `single-note`

Keep the default evidence at `conversation`; capture preserves supplied material without implying independent source review or verification.

## Recommended Note Shape

Use a short note with `Capture`, `Useful Points`, `Possible Links`, and `Follow-up`. Preserve the user's meaning, remove conversational filler, and avoid adding speculative explanation solely to make the note look complete.

## Append, Single-Note, and Split Criteria

- **Append** when a daily note, inbox note, or existing topic note is the established destination for this material.
- **Single-note** when the captured material has one retrieval purpose and needs no independent maintenance boundaries.
- **Split** only when the user supplied clearly separate items with different destinations; never create a multi-note knowledge expansion from an ordinary capture.

## Evidence Rules

Start and normally remain at `conversation`. Upgrade to `source-grounded` only when the active host actually reads attached or referenced material. Use `verified` only when an explicit validation step was also completed.

## Mode-Specific Validation

Verify that the capture is retrievable, faithful to the supplied material, free of fabricated context, and linked only to confirmed notes. Confirm that no unnecessary deep analysis or note proliferation was introduced.

## Completion Report Additions

Report the capture destination, whether content was appended or created, links added, follow-up retained, and any supplied material that could not be accessed.
