# Comparison Mode

## When to Use

Use `comparison` when choosing among tools, models, databases, frameworks, architectures, versions, or approaches against explicit needs. Do not use it for a list of unrelated product descriptions without a decision context.

## Required Inputs

Identify the options, decision owner, Requirements, constraints, Evaluation Dimensions, available evidence, unacceptable trade-offs, and whether a final decision or only a shortlist is expected.

## Default Knowledge Read

- Default depth: `standard`
- Default granularity: `single-note`

Keep the default evidence at `conversation`; popularity, familiarity, or mode choice never upgrades evidence by itself.

## Recommended Note Shape

Use `Decision Context`, `Requirements`, `Evaluation Dimensions`, `Evidence`, `Comparison Table`, `Trade-offs`, `Risks`, `Decision`, `Decision Rules`, and `Related Notes`. Apply the same dimensions and evidence standard to every option.

## Append, Single-Note, and Split Criteria

- **Append** when an existing decision note compares the same options for the same requirements and needs updated evidence.
- **Single-note** when one decision can be evaluated through a stable set of shared dimensions.
- **Split** when separate decisions have materially different requirements or owners; link their conclusions from a concise parent comparison.

## Evidence Rules

Start at `conversation`. Upgrade to `source-grounded` only after reading specifications, benchmarks, code, pricing, or primary documentation. Use `verified` only when representative tests or validated measurements confirm the compared claims.

## Mode-Specific Validation

Verify that each option is judged against identical Requirements and Evaluation Dimensions, Trade-offs are explicit, evidence dates and limits are visible, and the Decision follows from the stated priorities.

## Completion Report Additions

Report the options compared, decisive requirements, evidence gaps, rejected alternatives, decision status, and any benchmark or validation that was actually performed.
