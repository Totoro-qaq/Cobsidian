# Learning: Vector Databases

## Summary

Vector databases store embeddings and support similarity search for retrieval workflows.

## Core Concepts

- Embedding: numeric representation of text, images, or other data.
- Index: data structure used to make nearest-neighbor search faster.
- Recall: how many relevant items the search finds.
- Latency: how quickly search results return.

## Common Mistakes

- Treating vector search as a replacement for structured filters.
- Ignoring chunking quality.
- Evaluating only by demo output instead of fixed test queries.
